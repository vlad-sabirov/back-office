# Техническое задание: Ежедневник CRM — Фаза 1 MVP

## 1. Общее описание

Модуль "Ежедневник" — полноценный календарь с планированием дня и менеджером задач, интегрированный в CRM-систему Back Office. Включает в себя: менеджер задач (TaskManager), календарь событий (встречи, звонки, заметки, напоминания) и их отображение в едином интерфейсе.

---

## 2. Пользователи и роли

| Роль | Возможности |
|------|------------|
| **Сотрудник (менеджер)** | Ведёт свой ежедневник, создаёт события, задачи, заметки, напоминания. Видит свои задачи в календаре |
| **Руководитель** | Всё то же + просматривает ежедневники подчинённых, создаёт события/задачи в их календарях, назначает встречи |
| **Администратор (boss/developer/crmAdmin)** | Полный доступ ко всем ежедневникам |

---

## 3. Менеджер задач (TaskManager)

### 3.1 Описание
Система управления задачами с привязкой к CRM-организациям. Руководитель создаёт задачи сотрудникам, отслеживает выполнение, получает уведомления.

### 3.2 Модель данных — таблица: crm_task

| Поле | Тип | Описание |
|------|-----|----------|
| id | Int, PK | Автоинкремент |
| title | String | Название задачи |
| description | String? | Описание |
| status | String | "pending" / "in_progress" / "completed" / "cancelled" |
| priority | String | "low" / "normal" / "high" / "urgent" |
| deadline | DateTime? | Дедлайн |
| authorId | Int, FK → User | Кто создал задачу |
| assigneeId | Int, FK → User | Кому назначена |
| organizationId | Int?, FK → CrmOrganization | Привязка к организации |
| reminderSent3Days | Boolean | Напоминание за 3 дня отправлено |
| reminderSent1Day | Boolean | Напоминание за 1 день отправлено |
| reminderSent2Hours | Boolean | Напоминание за 2 часа отправлено |
| overdueNotified | Boolean | Уведомление о просрочке отправлено |
| createdAt | DateTime | Дата создания |
| updatedAt | DateTime | Дата обновления |
| completedAt | DateTime? | Дата завершения |

### 3.3 Модель данных — таблица: crm_task_modification

| Поле | Тип | Описание |
|------|-----|----------|
| id | Int, PK | Автоинкремент |
| taskId | Int, FK → CrmTask | Задача |
| modifiedById | Int, FK → User | Кто изменил |
| changes | String | JSON с описанием изменений |
| createdAt | DateTime | Дата изменения |

### 3.4 Функциональность задач

- CRUD операции (создание, чтение, обновление, удаление)
- Привязка к организации CRM
- Назначение исполнителя (assignee)
- Приоритеты: низкий, обычный, высокий, срочный
- Статусы: ожидает, в работе, выполнена, отменена
- История изменений (кто, когда, что изменил)
- Отображение задач в CRM-карточке организации (таб "Задача")

### 3.5 Права доступа к задачам

| Действие | Сотрудник | Руководитель | Админ |
|----------|-----------|-------------|-------|
| Создать задачу | В своих организациях | + | + |
| Просмотр задачи | Свои задачи | Свои + подчинённых | Все |
| Редактирование своей задачи (автор = сотрудник) | Все поля | Все поля | Все поля |
| Редактирование задачи от руководителя | Только статус | Свои задачи — все поля | Все поля |
| Удаление | Только свои (где автор) | Свои задачи | Все |

### 3.6 Telegram-уведомления задач

| Событие | Кому | Когда |
|---------|------|-------|
| Создание задачи | Исполнителю | Сразу |
| Изменение задачи (не самим исполнителем) | Исполнителю | Сразу |
| Напоминание за 3 дня до дедлайна | Исполнителю | Крон |
| Напоминание за 1 день до дедлайна | Исполнителю + руководителю | Крон |
| Напоминание за 2 часа до дедлайна | Исполнителю + руководителю | Крон |
| Задача просрочена | Исполнителю + руководителю | Крон |

| PATCH | `/crm/task/byId/:id/status` | Сменить статус |
| DELETE | `/crm/task/byId/:id` | Удалить задачу |

## 4. Типы событий календаря

### 4.1 Встреча
- Название, описание
- Дата + время начала и окончания
- Место (текст)
- Привязка к организации/контакту CRM (опционально)
- Назначается на сотрудника (assignee)

### 4.2 Запланированный звонок
- Название, описание
- Дата + время
- Привязка к организации/контакту CRM
- Краткое описание/цель звонка

### 4.3 Заметка
- Текст заметки
- Дата (событие на весь день)
- Привязка к организации/контакту CRM (опционально)

### 4.4 Напоминание
- Текст напоминания
- Дата + время срабатывания

| Поле | Тип | Описание |

| dateEnd | DateTime | Дата+время окончания |

### Индексы
- `(assigneeId, dateStart)` — основной запрос: события пользователя за период

- `(contactId)` — события контакта
- `(taskId)` — связь событие ↔ задача

## 6. API-эндпоинты календаря

Базовый путь: `/crm/calendar-event`

| Метод | URL | Назначение |
|-------|-----|-----------|
| POST | `/crm/calendar-event` | Создать событие |
| GET | `/crm/calendar-event/byId/:id` | Получить по ID |
| POST | `/crm/calendar-event/findMany` | Поиск с фильтрами |
| GET | `/crm/calendar-event/range?from=&to=&userId=` | События за период |
| GET | `/crm/calendar-event/range-with-tasks?from=&to=&userId=` | События + задачи для календаря |
| GET | `/crm/calendar-event/today-plan` | План на сегодня (для дашборда) |
| GET | `/crm/calendar-event/byOrganizationId/:id` | События организации |
| GET | `/crm/calendar-event/byContactId/:id` | События контакта |
| GET | `/crm/calendar-event/byTaskId/:id` | События привязанные к задаче |
| PATCH | `/crm/calendar-event/byId/:id` | Обновить событие |
| DELETE | `/crm/calendar-event/byId/:id` | Удалить событие |

### Ключевые эндпоинты

**`/range-with-tasks`** — основной эндпоинт для страницы календаря:
- Возвращает `{ events: CalendarEvent[], tasks: CrmTask[] }`
- Tasks фильтруются по `deadline` в указанном диапазоне и `status != completed/cancelled`
- Оба запроса выполняются параллельно через `Promise.all`

**`/today-plan`** — для виджета дашборда:
- Возвращает события и задачи на текущий день
- Автоматически определяет userId из JWT-токена

## 7. Права доступа к календарю

| Действие | Сотрудник | Руководитель | Админ |
|----------|-----------|-------------|-------|
| Создать своё событие | + | + | + |
| Создать событие подчинённому | - | + | + |
| Просмотр своего календаря | + | + | + |
| Просмотр календаря подчинённого | - | + | + |
| Редактирование своего события | + | + | + |
| Редактирование чужого события | - | Если автор | + |
| Удаление своего события | + | + | + |
| Удаление чужого события | - | Если автор | + |

Логика определения:
- `currentUserId === assigneeId` → свой календарь → полный доступ
- `User.child` содержит `assigneeId` → подчинённый → руководитель может просматривать и создавать
- Роли `boss/admin/developer/crmAdmin` → полный доступ ко всем

## 9. Интеграция задач и календаря

### 9.1 Отображение задач в календаре

| Действие | Как работает |
|----------|-------------|
| Задачи в календаре | Запрос CrmTask по deadline в диапазоне дат, отображение как событие |
| Создать задачу из календаря | Модальное окно → тип "Задача" → вызов API `POST /crm/task` |
| Создать задачу из CRM-карточки | Панель таба "Задача" → форма → вызов API `POST /crm/task` |
| Смена статуса из календаря | Вызов API `PATCH /crm/task/byId/:id/status` |

### 9.2 Привязка события календаря к задаче

Любое событие календаря (встреча, звонок, заметка, напоминание) может быть **опционально** привязано к задаче через поле `taskId`. К одной задаче можно привязать **несколько событий разных типов** — например, звонок + встреча + напоминание.

**Сценарий 1 — Создание задачи с несколькими событиями:**
При создании задачи (из CRM-карточки или из календаря) сотрудник/руководитель видит 4 независимых чекбокса:
- **"Добавить встречу"** — раскрывает поля: дата/время встречи, дата окончания, место
- **"Добавить звонок"** — раскрывает поля: дата/время звонка, цель/описание
- **"Добавить напоминание"** — раскрывает поля: текст, дата/время
- **"Добавить заметку"** — раскрывает поля: текст, дата

Чекбоксы **НЕ** взаимоисключающие — можно активировать любую комбинацию. После сохранения создаётся задача + N событий календаря (каждое с `taskId`). Запрос — последовательный: сначала `POST /crm/task`, затем `POST /crm/calendar-event` для каждого выбранного типа.

**Сценарий 2 — Добавление событий к существующей задаче:**
В карточке задачи (просмотр/редактирование) кнопка **"+ Добавить событие"** с выпадающим меню:
- **"Назначить встречу"** — модальное окно создания встречи
- **"Запланировать звонок"** — модальное окно создания звонка
- **"Добавить напоминание"** — модальное окно создания напоминания
- **"Добавить заметку"** — модальное окно создания заметки

Можно добавлять неограниченное количество событий к одной задаче. Предзаполнение:
- Организация (из задачи)
- Контакт (из организации, если один)
- Исполнитель (из задачи)
- `taskId` — автоматически

**Сценарий 3 — Привязка существующего события к задаче:**
В модальном окне редактирования любого события — поле "Связать с задачей" (Select/Autocomplete). Показывает задачи привязанной организации (или все задачи пользователя, если организация не выбрана).

**Сценарий 4 — Привязка задачи к существующему событию:**
В модальном окне редактирования любого события — поле "Связать с задачей". Позволяет выбрать задачу и связать.

### 9.3 Отображение связи

- В календаре: если событие привязано к задаче — иконка-связь и статус задачи, иконка типа (телефон/встреча/колокольчик/заметка)
- В карточке задачи: блок **"Связанные события"** — список всех привязанных событий, сгруппированных по типу (встречи, звонки, напоминания, заметки) с датами и иконками. Счётчик: "3 события"
- В CRM-карточке организации: таб "Событие" показывает привязку к задаче (если есть)
- В виджете "План на сегодня": события привязанные к задачам отображаются с пометкой задачи

### 9.4 API для связи

| Метод | URL | Назначение |
|-------|-----|-----------|
| GET | `/crm/calendar-event/byTaskId/:id` | События привязанные к задаче |
| PATCH | `/crm/calendar-event/byId/:id` | Обновление taskId (привязка/отвязка) |

При удалении задачи — привязанные события НЕ удаляются, поле `taskId` обнуляется (SET NULL).
