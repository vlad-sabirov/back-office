# План действий по улучшению UX и удобства CRM

## 1. Критично исправить

### 1.1. Привести карточку организации к одной понятной структуре
- Убрать разрыв между заметками и остальными активностями.
- Решить, что является основной лентой активности: комментарии, заметки, задачи, события.
- Либо встроить заметки в табы, либо объединить комментарии и заметки в единый сценарий.
- Снизить визуальную перегрузку карточки и убрать дублирующиеся паттерны взаимодействия.

### 1.2. Переделать логику фильтра организаций
- Убрать скрытый сброс большого числа полей при переключении табов.
- Разделить пресеты и ручные фильтры.
- Сделать состояние фильтра предсказуемым для пользователя.
- Явно показывать, какие фильтры сейчас применены.

### 1.3. Довести ежедневник до честного рабочего сценария
- Либо реализовать полноценные режимы `день / неделя / месяц`.
- Либо сузить позиционирование страницы, если пока поддерживается только месяц.
- Упростить сценарий просмотра своего календаря и календаря подчинённых.
- Отдельно проверить читаемость задач и событий в одном календаре.

### 1.4. Стабилизировать виджет "Мой день"
- Перенести сбор данных в единый backend-эндпоинт.
- Исключить несинхронную загрузку разных блоков.
- Сделать одну понятную рабочую сводку: просроченные, на сегодня, порученные, остывающие организации.
- Упростить визуальную иерархию секций.

## 2. Быстрые улучшения с высоким эффектом

### 2.1. Ограничить риск ошибочных быстрых действий
- Для быстрых действий по организации добавить подтверждение ключевых полей.
- Не подставлять без подтверждения первого попавшегося контакта или телефона.
- Явно показывать, на кого будет назначена задача или событие.
- Сделать время дедлайна/события редактируемым и очевидным.

### 2.2. Улучшить UX авторизации
- Сделать сообщения об ошибках более понятными.
- Явно описать сценарий первого входа.
- Упростить переход между шагами логина и PIN/Telegram-подтверждением.
- Добавить понятные подсказки, если бот отключён или пользователь уволен.

### 2.3. Улучшить пустые состояния
- Добавить полезные `empty state` в ежедневник, CRM-виджеты и новые блоки.
- Показывать не просто "ничего нет", а следующее полезное действие.
- Для новых сущностей добавить короткие пояснения, зачем они нужны.

## 3. Функциональные улучшения продукта

### 3.1. Добавить единый центр внимания менеджера
- Собрать в одном месте всё, что требует действия сейчас.
- Включить просроченные задачи, новые события, поручения и проблемные организации.
- Дать быстрый переход к нужному объекту без лишних кликов.

### 3.2. Сделать внутренний центр уведомлений
- Не опираться только на Telegram.
- Добавить уведомления внутри интерфейса.
- Отделить информационные уведомления от действительно срочных.

### 3.3. Добавить глобальный поиск по CRM
- Поиск по организациям, контактам, задачам, событиям.
- Быстрый переход к карточке и связанным действиям.

## 4. Техническая дисциплина

### 4.1. Закрепить правило по Prisma
- Любое изменение `schema.prisma` должно сопровождаться миграцией.
- Проверять это перед коммитом и перед деплоем.

### 4.2. Ввести UX-проверку перед выпуском
- Проверять новые сценарии не только на компиляцию, но и на удобство.
- Делать короткий чек-лист: понятность, количество кликов, предсказуемость, ошибки, пустые состояния.

## 5. Рекомендуемый порядок работ

1. Карточка организации: заметки, комментарии, активности.
2. Фильтр организаций: пресеты и прозрачное состояние.
3. Виджет "Мой день": единый источник данных и упрощение структуры.
4. Ежедневник: честный сценарий day/week/month или явное упрощение.
5. Быстрые действия в таблице организаций.
6. UX авторизации.
7. Внутренние уведомления и глобальный поиск.
